\begin{console}
df
\end{console}

For F31, user files are probably stored in \verb!/dev/mapper/fedora-root!
which should be in the \verb!/dev/sda2/! partition.
For f40, user files are probably stored in \verb!/dev/nvme0n1p3!.
Run \verb!su! to become root and then run \verb!gparted! and you'll see the
following partitions of your HD:
\begin{center}
\includegraphics[width=4in]{gp0.png}
\end{center}
Note the biggest partition is about 19GB and take note the name.

To expand your HD, do the following:

\begin{enumerate}
\li Shutdown your virtual machine.
\li Run VMware workstation. 
\li Click on Hard Disk. (For F31 next to the Hard Disk is SCSI.
For F40 you see NVMe.)
You'll see \lq\lq Capacity: Maximum size 20GB".
\li (You might want to click on Compact.)
\li Click on \lq\lq Expand" and then choose Maximum disk size (say 100.0)
\li Run your virtual machine.
\end{enumerate}

In your bash shell, run \verb!su! to be root.
Run \verb!gparted!:
\begin{center}
\includegraphics[width=4in]{gp1.png}
\end{center}
Right click on \verb!/dev/nvme0n1p3! (the largest partition)
and select Resize/Move:
\begin{center}
\includegraphics[width=4in]{gp2.png}
\end{center}
Make this partition as large as possible:
\begin{center}
\includegraphics[width=4in]{gp3.png}
\end{center}
Click on the green arrow and Apply All Operations.

Run \verb!gparted! again and you'll see:
\begin{center}
\includegraphics[width=4in]{gp4.png}
\end{center}
Note the size of the largest partition is now largest.

For F31, you'll also need to do the following.
In your bash shell, run
\begin{console}
lvm lvresize --verbose --resizefs -L 70G /dev/mapper/fedora-root
\end{console}
replacing 70G with whatever is appropriate.
Run
\begin{console}
df
\end{console}
again and check that 
\verb!/dev/mapper/fedora-root! is larger.
